**1. [Rule 1 - Bug] Fixed DrizzleAdapter emailVerified type conflict**
- **Found during:** Task 3 (emailVerified JWT claim)
- **Issue:** DrizzleAdapter expects `emailVerified: Date | null` but code was conflicting with boolean JWT claim
- **Fix:** Kept DB column as timestamp (Date), boolean conversion only happens in JWT callback via `dbUser?.emailVerified != null`
- **Files modified:** apps/web/src/lib/auth.ts

**2. [Rule 1 - Bug] Fixed 0007_snapshot.json: stale autoincrement fields + UUID collision**
- **Found during:** Task 5 (drizzle-kit generate)
- **Issue:** drizzle-kit 0.31.10 column2.strict() rejects `autoincrement` field (removed in newer versions). 8 columns in 0007 snapshot had it. Also 0006 and 0007 had identical id and prevId UUIDs, causing a "collision" error.
- **Fix:** (1) Stripped `autoincrement` from all 8 affected columns in 0007_snapshot.json. (2) Assigned a fresh UUID to 0007.id and set 0007.prevId = 0006.id to restore the correct chain.
- **Files modified:** apps/web/db/migrations/meta/0007_snapshot.json
- **Verification:** drizzle-kit generate succeeded, outputting 0008_silent_susan_delgado.sql
- **Committed in:** 02df30a
