## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → /onboarding/cadastral | authenticated user; mandatory step |
| client → PATCH /api/user/org-profile | authenticated user; org row update + Asaas sync |
| proxy.ts → JWT claim | trusted (signed by NextAuth); read at edge |
| api/user/org-profile → Asaas API | trusted server-side fetch with ASAAS_API_KEY |
| RSC /flows/[id] → DB | scoped by orgId; n8nExecutionId join must respect tenant |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Spoofing / Elevation | User manipulates JWT to skip CNPJ guard | mitigate | orgCnpjFilled is computed server-side in NextAuth jwt callback from DB — NextAuth signs the JWT; client cannot tamper. Guard re-reads via session, never trusts request body |
| T-02-01b | Bypass | Direct request to /api/user/org-profile to bypass cadastral | accept | This endpoint IS the bypass — it's how the user fills the data. After successful PATCH, JWT refresh on next session rotation re-evaluates orgCnpjFilled |
