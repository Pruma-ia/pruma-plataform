## Component Contracts

### 1. OTP Input — `/verify-email`

**Surface:** auth panel (dark glass, same as login/register pages)

**Layout:**
```
Auth layout (full-screen gradient background)
└── Panel: rounded-2xl border-white/10 bg-white/5 backdrop-blur-sm p-8 shadow-2xl max-w-sm
    ├── Logo: /logo-white.png h-9 w-auto mb-7 centered
    ├── Title: text-xl font-semibold text-white mb-1
    ├── Subtitle: text-sm text-white/60 mb-6 (shows truncated email)
    ├── 6 OTP digit cells (row, gap-2)
    │   └── Each cell: w-11 h-13 rounded-lg border border-white/20 bg-white/10
    │       text-center text-xl font-semibold text-white
    │       focus:ring-2 focus:ring-[#00AEEF]/70 focus:outline-none
    │       caret-transparent
    ├── Error: role="alert" text-sm text-red-400 mt-2 (conditionally shown)
    ├── Primary button: full-width bg-[#00AEEF] py-2.5 text-sm font-semibold text-white
    │   rounded-lg hover:bg-[#00AEEF]/90 disabled:opacity-60
    └── Resend row: text-xs text-white/40 mt-4 centered
        ├── Available: button text-[#5CCFF5] hover:text-white (active after 60s)
        └── Cooldown: text-white/40 + countdown "{N}s" in text-[#00AEEF]/70
```

**OTP input behavior:**
- 6 separate `<input type="text" inputMode="numeric" maxLength={1}>` cells
- Auto-advance: on valid digit entry, focus moves to next cell
- Auto-retreat: on Backspace when cell empty, focus moves to previous cell
- Paste support: distributes pasted 6-digit string across cells
- Only accepts digits 0–9; non-digit keystrokes ignored
- All 6 cells must be filled before "Verificar código" button is enabled
- Button disabled + spinner while POST in-flight
- On error response: clear all cells, focus first cell, show error message
- On success: button shows spinner briefly, then page redirects

**Resend button state machine:**
- State A — cooldown (0–60s after page load or last resend): shows `Reenviar em {N}s`, non-clickable, color `text-white/40`
- State B — available (after 60s): shows `Reenviar código`, clickable, color `text-[#5CCFF5]`
- State C — in-flight (during POST /api/auth/resend-otp): shows `Enviando...`, disabled
- Countdown is a JS interval that counts down from 60 to 0 in UI only; actual cooldown enforced server-side

**Accessibility:**
- `<fieldset>` wrapping 6 inputs with `<legend className="sr-only">Código de verificação</legend>`
- Each input: `aria-label="Dígito {N} de 6"` where N is 1–6
- Error `<p role="alert">` — always rendered in DOM, content conditionally set (avoids layout shift)
- Submit button: `aria-busy="true"` when loading

**Upload flow behavior:**
1. User drops or selects file — client validates MIME (PNG/JPG/WebP) and size (≤2MB) immediately
2. Client error → show error message, no upload initiated
3. Client valid → show uploading state → POST to `/api/organizations/logo/presign` → receive `{ uploadUrl, r2Key }`
4. Browser PUT to `uploadUrl` directly → on success → PATCH `/api/organizations/profile` with `{ logo: r2Key }`
5. On PATCH success → show preview, toast `Logo atualizada com sucesso.`
6. On any failure → show error message, return to idle state

**Hidden file input:** `<input type="file" accept="image/png,image/jpeg,image/webp" className="sr-only" />`

**Fallback initials avatar (shown when logo is null):**
- Circle `rounded-full bg-primary` size 80×80
- Initials: `text-primary-foreground text-xl font-semibold font-heading`
- Logic: split org name by spaces, take first char of first 2 words, uppercase

**Accessibility:**
- Upload zone: `role="button" tabIndex={0}` with `onKeyDown` (Enter/Space triggers file picker)
- `aria-label="Carregar logo da organização"` on the zone
- File input: `aria-hidden="true"` (triggered programmatically)

## Interaction Patterns Summary

| Interaction | Pattern |
|-------------|---------|
| Button loading | `disabled` + `<Loader2 className="animate-spin h-4 w-4" />` + label change |
| Toast success | `duration: 3000` auto-dismiss |
| Toast error | `duration: 5000` manual dismiss |
| Focus ring | `focus-visible:ring-2 focus-visible:ring-ring` (Ciano Elétrico) — never removed |
| Hover on button | `transition-colors duration-150` |
| Hover on stat card | `hover:shadow-md transition-shadow` |
| Drag-over on upload zone | `border-accent bg-secondary ring-2 ring-accent/30` |
| Error display timing | On blur (form fields) — not on keystroke |
| Error `role="alert"` | Always present in DOM, content toggled |
| Destructive confirm | Dialog required — no in-Phase-1 destructive actions except logo remove (see below) |
| Logo remove | `variant="ghost"` button with `text-destructive` — single click, no confirm dialog (low-risk, re-uploadable) |

## Accessibility Checklist

All new components must satisfy:

- [ ] Every `<label>` has `htmlFor` paired to input `id`
- [ ] All `autoComplete` values set semantically (`name`, `email`, `one-time-code` for OTP)
- [ ] Dynamic errors use `role="alert"`
- [ ] Focus rings never suppressed (`outline-none` only paired with `ring-*` replacement)
- [ ] Interactive non-button elements have `role="button"` + `tabIndex={0}` + keyboard handler
- [ ] Images have descriptive `alt` text
- [ ] Icon-only buttons have `aria-label`
- [ ] Minimum contrast: 4.5:1 for normal text, 3:1 for large text
- [ ] OTP inputs: `inputMode="numeric"` + `autoComplete="one-time-code"` on first cell
