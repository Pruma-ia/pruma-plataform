## Componentes por Tela

### 1. `/approvals` — Listagem com Filtros, Busca e Export

**Origem:** refactor do `ApprovalsList` + `ApprovalsPage` existentes. As pills de status já existem; esta fase adiciona busca textual, filtro por fluxo, filtro por data, export CSV e paginação server-side.

#### Toolbar (nova)

```
flex flex-col gap-3 mb-4
  linha 1: flex items-center gap-2 flex-wrap
    input de busca — w-64 — ícone Search à esquerda (pl-9)
    select "Fluxo" — w-48
    input date "De" — w-36
    input date "Até" — w-36
    (se filtros ativos) Button outline sm "Limpar filtros"
  linha 2: flex items-center justify-between
    pills de status (existentes: Total / Pendentes / Aprovadas / Rejeitadas)
    Button outline sm "Exportar CSV" ícone Download
```

**Input de busca:**
- Largura: `w-64`
- Ícone: `Search` Lucide `h-4 w-4 text-muted-foreground`, posicionado à esquerda com `pl-9`
- Debounce: 300ms antes de `router.push(?q=...)`
- Placeholder: "Buscar aprovação..."
- `aria-label="Buscar aprovações"`, `autoComplete="off"`

**Selects e inputs de filtro:**
- Classe base: `rounded-md border border-input bg-background text-sm px-3 py-2 h-9`
- Select de fluxo: opção vazia "Todos os fluxos" + lista de fluxos da org
- Datas: `<input type="date">` com mesma classe base
- `htmlFor` + `id` em todo par label/input (label pode ser `sr-only` quando contexto é claro)

**Botão export:**
- Variante: `outline`, tamanho `sm`
- Label: "Exportar CSV"
- Ícone: `Download` Lucide à esquerda
- Implementado como `<a href="/api/approvals/export?{params}" download>` — tag nativa, sem JS de fetch

#### Tabela

Colunas: Status (ícone) | Título + Descrição | Fluxo | Data criação | Resolvido por | Anexos | Ação

- Coluna "Fluxo" nova entre Título e Data: `text-xs text-muted-foreground`, `hidden md:table-cell`
- Paginação no rodapé da tabela (nova):

```
flex items-center justify-between border-t px-4 py-3
  "Mostrando X–Y de Z aprovações" — text-xs text-muted-foreground
  flex gap-1
    Button ghost sm "Anterior" — disabled na página 1
    Button ghost sm "Próxima" — disabled na última página
```

#### Estados obrigatórios

| Estado | Estrutura |
|---|---|
| Loading | Skeleton com 5 linhas — mesma estrutura da tabela real, `animate-pulse` |
| Vazio — sem aprovações ainda | Ícone `ClipboardList` + "Nenhuma aprovação recebida ainda" + "Quando o n8n criar aprovações via webhook elas aparecerão aqui." |
| Vazio — filtros sem resultado | Ícone `Search` + "Nenhum resultado" + "Tente ajustar os filtros ou limpar a busca." + Button outline "Limpar filtros" |

## Padrões de Interação

Fonte: `design-system/MASTER.md`.

| Evento | Comportamento |
|---|---|
| Hover em botão | `transition-colors duration-150` |
| Focus em input | `ring-2 ring-ring` (Ciano Elétrico) — nunca `outline-none` sem substituto |
| Submit de form | Botão `disabled` + `Loader2 animate-spin` inline |
| Input de busca | Debounce 300ms → `router.push(?q=...)` — sem botão de submit |
| Export CSV | `<a href="/api/approvals/export?{params}" download>` — tag nativa |
| Select de fluxo / input de data | `onChange` → `router.push` imediato — sem botão "Aplicar" |
| Pill de status | `onClick` → `router.push(?status=...)` — migrar lógica client-state atual para URL param |
| Paginação Anterior / Próxima | `router.push(?page=N)` |
| CEP `onBlur` | Fetch ViaCEP → preenche campos automaticamente |
| Redirect pós-onboarding | `window.location.href = "/dashboard"` (hard reload para forçar re-execução do JWT callback — padrão do step anterior) |

**Regra de URL params:** todos os filtros de `/approvals` vivem em URL search params. Usar `new URLSearchParams(searchParams)` + `.set/.delete` individual — nunca sobrescrever o objeto inteiro de uma vez.

## Acessibilidade

Fonte: `design-system/MASTER.md` + `apps/web/CLAUDE.md` (Convenções de Formulários).

- `htmlFor` + `id` em todo par label/input — obrigatório inclusive nos filtros
- `aria-label="Buscar aprovações"` no input de busca (sem label visível)
- `role="alert"` em mensagens de erro dinâmicas (CNPJ inválido, erro de submit, erros de campo)
- `aria-describedby` nos campos com helper text visível (ex: "Buscando endereço...")
- `aria-invalid` nos inputs com estado de erro
- `autoComplete` semântico: CNPJ → `"off"`, telefone → `"tel"`, CEP → `"postal-code"`, cidade → `"address-level2"`, UF → `"address-level1"`
- Timeline: `<ol>` com `<li>` — lista ordenada de eventos de auditoria
- Tabela de runs: `<caption className="sr-only">Execuções recentes do fluxo</caption>`
- Botões de paginação: `aria-label="Página anterior"` / `aria-label="Próxima página"` + `disabled` semântico
- Contraste mínimo 4.5:1 (texto normal), 3:1 (texto grande) — garantido pelos tokens semânticos oklch
